## 一、项目概览

### 1.1 项目理念

灵犀学院是一款开源的引导式 AI 学习应用，核心理念包含三点：

1. **非商业化** —— 项目以 MIT 协议开源，不内置任何付费墙或广告
2. **用户自备 API** —— 应用本身不分发 AI 服务，用户需自行配置 API Key（OpenAI / Claude / Gemini / Ollama），API Key 仅本地加密存储，**永不上传**
3. **苏格拉底式引导** —— AI 导师以提问引导思考而非直接给出答案，培养独立思考与问题拆解能力

### 1.2 核心功能特性

| 功能 | 说明 |
|------|------|
| 🤔 引导式学习 | 苏格拉底式问答，AI 不给答案给引导 |
| 💬 自由对话 | 支持 OpenAI / Claude / Gemini / Ollama 四类 Provider |
| 📚 学习路径 L0-L4 | 结构化知识点卡片 + 测验 + 苏格拉底对话 |
| 🦏 吉祥物小犀 | 6 状态机情绪联动，点击交互，连点 5 次触发彩蛋 |
| 🔥 Streak 连续打卡 | 连续学习天数追踪 + 成就徽章系统 |
| 🔍 分级探索 | 简化 / 深入 / 图示三按钮，按需调整 AI 回复详略 |
| 🔒 数据安全 | API Key 硬件级加密存储，日志自动脱敏 |
| 🖥️ 双端支持 | Android + Windows 跨平台 |

### 1.3 学习路径层级

| 级别 | 定位 | 内容方向 |
|------|------|----------|
| L0 | 启蒙 | AI 基础概念与工具入门 |
| L1 | 基础 | 编程语言与提示词工程基础 |
| L2 | 进阶 | 机器学习与数据处理 |
| L3 | 实践 | 深度学习与模型训练 |
| L4 | 应用 | AI 项目实战与部署 |

每个知识点遵循闭环：**知识卡片学习 → 测验检验 → 苏格拉底式对话深化**，完成即解锁下一节点。

## 二、整体架构

### 2.1 分层架构

灵犀学院采用**严格分层架构**，依赖方向单向向下：

```
┌─────────────────────────────────┐
│         UI Layer (features/)     │   页面、组件、交互逻辑
├─────────────────────────────────┤
│     State Management (Riverpod)  │   Provider / StateNotifier
├─────────────────────────────────┤
│      Business Logic (Services)   │   成就服务、连续学习、数据导出
├─────────────────────────────────┤
│    Data Access (Repositories)    │   每表一个 Repository，封装 CRUD
├─────────────────────────────────┤
│      Storage (Drift + Secure)    │   SQLite 数据库 + SecureStorage
└─────────────────────────────────┘
```

### 2.2 各层职责

| 层 | 目录 | 职责 | 依赖方向 |
|----|------|------|----------|
| **UI 层** | `lib/features/` | 页面 Widget、业务编排、交互响应 | 依赖 State / Data / Shared |
| **状态层** | Riverpod Providers | 跨组件状态共享、依赖注入 | 依赖 Data 层 |
| **业务层** | `lib/features/*/services/` | 成就解锁、Streak 计算、数据导出 | 依赖 Data 层 |
| **数据层** | `lib/data/repositories/` | 表 CRUD、数据模型转换 | 依赖 Storage 层 |
| **存储层** | `lib/data/db/` + `lib/data/services/` | Drift SQLite + flutter_secure_storage | 只依赖第三方库 |

### 2.3 分层原则

1. **依赖方向单向向下**：上层可依赖下层，下层不可反向依赖上层
2. **core 层无业务逻辑**：`lib/core/` 只含主题、路由、常量、动画曲线等基础设施
3. **shared 层不依赖 feature**：`lib/shared/` 是跨 feature 复用的纯 UI 组件与工具
4. **业务编排集中在 features**：Controller（StateNotifier）负责调用 Repository / Service 完成业务流程

### 2.4 状态管理策略

项目使用 `flutter_riverpod` 2.5.x，所有跨组件状态必须通过 Riverpod 管理：

| Provider 类型 | 适用场景 | 项目示例 |
|--------------|----------|----------|
| `Provider` | 无状态依赖/服务/单例 | `databaseProvider`、`secureStorageServiceProvider` |
| `StateProvider` | 简单可变状态 | `themeModeProvider`、`localeProvider`、`socraticModeProvider` |
| `FutureProvider` | 异步一次性数据 | `currentAiProviderProvider`、`promptManagerProvider` |
| `StateNotifierProvider` | 复杂状态机 | `chatControllerProvider`、`mascotControllerProvider` |

> **使用约定**：
> - Repository / Service / 数据库连接等单例 Provider **不加** autoDispose
> - `ref.watch` 在 `build` 内订阅状态变化触发重建
> - `ref.read` 在回调 / 事件处理 / Controller 方法内一次性读取

### 4.4 router/app_router.dart

#### `goRouterProvider` (Provider<GoRouter>)

全局路由配置，包含：

1. **redirect 逻辑（onboarding 引导）**：
   - 读取 `SharedPreferences.onboarding_completed`
   - 未完成引导：除 `/onboarding`、`/onboarding/api-setup`、`/settings/api` 外全部重定向到 `/onboarding`
   - 已完成但访问引导页：重定向回 `/home`

2. **路由树结构**：
   - 顶层路由：`/onboarding`、`/onboarding/api-setup`
   - `ShellRoute`（导航壳）：包裹所有主导航页面
     - `/home`、`/learning`（含子路由 `:courseId/:lessonId`）、`/chat`（含 `:conversationId`）、`/notes`（含 `:noteId`）、`/achievements`、`/statistics`、`/settings`、`/settings/api`、`/help`

3. **`_AppShell`（导航壳 Widget）**：
   - 桌面端（≥1024px）：`NavigationRail` + `VerticalDivider`
   - 移动端：`NavigationBar`
   - 6 个主导航项：首页 / 学习 / 对话 / 笔记 / 成就 / 设置

### 4.5 theme/

`lib/core/theme/` 由 5 个文件协同构成完整的双主题（light/dark）系统，所有视觉令牌（颜色 / 渐变 / 阴影 / 形状）均通过 `ThemeExtension` 注入 `ThemeData.extensions`，UI 层统一通过 `context.lingxiColors` / `context.lingxiGradients` / `context.lingxiElevations` 扩展获取，**禁止**硬编码颜色值。

| 文件 | 职责 |
|------|------|
| `app_theme.dart` | `AppTheme` 静态类，提供 `lightTheme` / `darkTheme`，种子色 `0xFF6750A4`（紫色调），Noto Sans SC + Quicksand 字体；`darkTheme` 中通过 `brightness` 三元切换注册 `LingxiColors.dark` / `LingxiGradients.dark` / `LingxiElevations.dark` 三组 ThemeExtension |
| `lingxi_colors.dart` | `LingxiColors extends ThemeExtension<LingxiColors>`，6 个语义色（mascotPrimary 星空紫 / streakFire 火焰红 / achievementGold 成就金 / misconceptionRed 误解红 / successGreen 成功绿 / infoBlue 信息蓝），提供 `light` / `dark` 双静态实例；`dark` 实例校准 streakFire（0xFFFF7043→0xFFFF8A65）与 achievementGold（0xFFFFE082→0xFFFFD54F）以满足 WCAG AA 对比度（≥ 4.5:1） |
| `lingxi_gradients.dart` | `LingxiGradients extends ThemeExtension<LingxiGradients>`，6 个语义渐变（mascotHero 吉祥物主光 / streakFire 火焰 / achievementGold 成就金 / primarySurface 主色面 / celebration 庆祝 / success 成功），提供 `light` / `dark` 双静态实例 |
| `lingxi_elevations.dart` | `LingxiElevations extends ThemeExtension<LingxiElevations>`，重构为 3 档语义阴影 `subtle` / `elevated` / `highlighted`（light/dark 双实例），同时保留 `level0`~`level4` 兼容旧调用；`LingxiCard.elevation` 参数映射 0→subtle / 1→elevated / 2→highlighted |
| `shape_variants.dart` | 形状变体定义 |

#### 4.6.1 Hero 共享元素动画（v0.3.0 新增）

吉祥物在首页 / 学习路径 / 对话页之间通过 Hero 共享元素动画实现视觉延续。

| 组件 / API | 位置 | 职责 |
|------------|------|------|
| `Hero(tag: 'mascot-hero')` | `lib/features/home/home_page.dart` / `lib/features/learning/learning_path_page.dart` / `lib/features/chat/chat_page.dart` | 三处吉祥物外层包裹同名 Hero，tag 统一为 `'mascot-hero'` |
| `mascotHeroFlightShuttleBuilder` | `lib/features/mascot/` | 自定义 `flightShuttleBuilder`，飞行过程使用 `SpringMotion.gentleSpeed` 曲线插值位置与缩放，保证双端过渡手感一致 |
| `reduceMotion` 降级 | `MediaQuery.disableAnimationsOf(context)` 为 true 时 | Hero 动画降级为即时切换，跳过飞行过程 |

**使用约束**：同屏不可出现两个相同 tag 的 Hero，否则抛 `MultipleHeroException`。三处页面均为单吉祥物场景，已通过静态审查。

#### 4.6.2 按压微交互（v0.3.0 新增）

通用组件统一接入按压弹性反馈，所有按压动画在 `SpringMotion.fastSpeed`（148ms）下完成。

| 组件 | 文件 | 按压效果 | 触发条件 |
|------|------|----------|----------|
| `LingxiButton` | `lib/shared/widgets/lingxi_button.dart` | `AnimatedScale` 0.96 + 阴影从 `subtle` 抬升至 `elevated` | `onPressed != null` 且按压时 |
| `LingxiCard` | `lib/shared/widgets/lingxi_card.dart` | `AnimatedScale` 0.99 + 阴影抬升 | 仅可点击卡片（`onTap != null`）触发，不可点击卡片无 `AnimatedScale` |
| `LingxiChip` | `lib/shared/widgets/lingxi_chip.dart` | 选中态 `AnimatedSwitcher` + scale 0.95 | 选中态切换时触发 |

**降级**：`reduceMotion` 为 true 时，按压动画跳过 `AnimatedScale`，仅保留阴影抬升的瞬时切换。

#### 4.6.3 GoRouter 页面过渡统一（v0.3.0 升级）

所有 14 个 `GoRoute`（onboarding / apiSetup / home / learning / lesson / chatList / chat / notes / noteEditor / achievements / statistics / settings / settingsApi / help）统一通过 `LingxiPageTransitions.buildPage` / `buildSlidePage` / `buildModalPage` 返回 `CustomTransitionPage`。

| 要素 | 实现 |
|------|------|
| 过渡 builder | `LingxiPageTransitions.slideFadeTransitionBuilder`：`FadeTransition` + `SlideTransition`，begin `Offset(0.0, 0.05)` → `Offset.zero` |
| 入场曲线 | `SpringMotion.entranceCurve` |
| 出场曲线 | `SpringMotion.exitCurve` |
| 时长常量 | `slowDuration` / `defaultDuration` / `gentleDuration`（取自 `SpringMotion` 6 档规格） |
| 双端一致性 | Android / Windows 共享同一套常量与曲线，无平台分支 |
| `reduceMotion` 降级 | `slideFadeTransitionBuilder` / `_buildSlideTransition` / `_buildModalTransition` 在 `MediaQuery.disableAnimationsOf` / `SpringMotion.reduceMotionOf` 为 true 时直接返回 `child`（即时切换） |

**ShellRoute 兼容**：导航壳内子页面过渡未破坏 `_AppShell` 的 NavigationRail / NavigationBar 切换。

### 6.1 ai/ —— AI Provider 抽象与实现

#### `ai_provider.dart`（抽象接口）

定义所有 AI 服务商必须遵循的统一接口：

```dart
abstract class AiProvider {
  Stream<AiStreamEvent> chatStream({
    required List<ChatMessage> messages,
    required ChatOptions options,
  });
  Future<String> chat({...});
  void cancel();
  Future<bool> testConnection();
}
```

**关键数据模型**：

| 类型 | 说明 |
|------|------|
| `enum MessageRole` | system / user / assistant |
| `class ChatMessage` | 不可变值对象（role + content） |
| `class ChatOptions` | temperature / maxTokens / systemPrompt / stream |
| `sealed class AiStreamEvent` | 流式响应事件基类 |
| `TextDeltaEvent` | 内容增量 |
| `UsageEvent` | token 用量统计 |
| `DoneEvent` | 流式结束 |
| `ErrorEvent` | 流式错误 |

**契约**：流必须以 `DoneEvent` 或 `ErrorEvent` 终止；被 `cancel` 时静默结束不抛异常。

#### `ai_provider_registry.dart`

AI Provider 注册中心，有状态单例：

| 方法 | 职责 |
|------|------|
| `getProvider()` | 获取当前 Provider，缓存命中直接返回；否则从仓库读取默认配置创建 |
| `_createProvider(config)` | 根据 ProviderType switch 创建对应实现 |
| `switchProvider(type)` | 切换到指定类型 |
| `invalidateCache()` | 失效缓存（用户修改配置后调用） |
| `cancel()` | 取消当前进行中的请求 |

#### `ai_providers.dart`（Riverpod 注册）

| Provider | 类型 | 职责 |
|---------|------|------|
| `aiProviderRegistryProvider` | `Provider<AiProviderRegistry>` | 单例，依赖 `providerConfigRepositoryProvider` |

#### 四个具体 Provider 实现

| 文件 | 类 | 协议 | 鉴权 |
|------|----|----|------|
| `openai_compatible_provider.dart` | `OpenAICompatibleProvider` | POST `/chat/completions` + SSE | Bearer Token |

W3C SSE 规范解析器：

### 6.11 mascot/ —— 吉祥物小犀

#### `mascot_state.dart`

```dart
enum MascotMood { idle, happy, thinking, sad, celebrate, curious }

class MascotState {
  final MascotMood mood;
  final bool isAnimating;
  final int tapCount;
}
```

**状态优先级**：sad > celebrate > thinking > happy > curious > idle

#### `mascot_controller.dart`

`MascotController extends StateNotifier<MascotState>`：

| 方法 | 职责 |
|------|------|
| `setMood(mood)` | 直接设定情绪，标记动画进行中 |
| `setAiThinking(thinking)` | 联动 AI 思考态 |
| `triggerTap()` | 2 秒内连点 5 次→celebrate 3s；否则 happy 1.5s |
| `celebrate()` | 触发庆祝动画 3s 后恢复 |
| `reset()` | 重置为初始待机 |

**Provider**：`mascotControllerProvider`（StateNotifierProvider，跨页面共享）

#### 其他 mascot 文件

| 文件 | 类 | 职责 |
|------|----|------|
| `mascot_widget.dart` | `MascotWidget` + `_MascotPainter`（内部） | 主 Widget，含光环 `_AuraGlow` 与气泡 `_SpeechBubble`，点击触发 `triggerTap()`，内部封装 `_MascotPainter` 作为 fallback 绘制 |
| `rive_mascot_widget.dart` | `RiveMascotWidget` | 目标方案，加载 `assets/rive/lingxi_mascot.riv`，加载失败降级为 `MascotWidget`（不再直接引用 `MascotPainter`） |
| `mascot_overlay.dart` | `MascotOverlay` | 全局浮层，mood != idle 时在右下角悬浮显示（72px 缩略版） |

#### `_MascotPainter` 视觉实现细节

`_MascotPainter`（`mascot_widget.dart` 内部私有类）作为 Rive 资源未就绪时的矢量 fallback 绘制器，已升级为具有完整视觉表现的实现：

| 视觉元素 | 实现 |
|---------|------|
| 身体填充 | 径向渐变（`LingxiColors.mascotPrimary` → 深紫 #5E35B1），营造立体感 |
| 角部高光 | 高光弧线点缀，强化"小犀"角色识别 |
| 眼睛 | 瞳孔高光点（白色小圆），增加灵性 |

6 种情绪差异化绘制：

| 情绪 | 视觉表现 |
|------|---------|
| `idle` | 眨眼动画 + 瞳孔高光点 |
| `happy` | 弧形笑眼 + 跳跃偏移 |
| `thinking` | 头顶问号浮动呼吸（缩放 + 透明度） |
| `sad` | 泪滴滑落动画（纵向位移 + 渐隐） |
| `celebrate` | 10 粒子 × 4 段拖尾辐射，圆周分布 |
| `curious` | 放大镜光泽弧线（旋转 + 透明度脉动） |

**`shouldRepaint` 优化**：仅在 `mood` / `animationValue` / `extraSparkle` 变化时返回 `true`，避免无谓重绘。

#### `_AuraGlow` 光环实现

`_AuraGlow`（`mascot_widget.dart` 内部私有 Widget）为吉祥物提供呼吸光环背景：

- 渐变：使用 `LingxiGradients.mascotHero` 渐变（随主题切换 light/dark 实例）
- 动画：4 秒呼吸脉动（scale 0.95 → 1.05），`Curves.easeInOut`
- 性能：`RepaintBoundary` 隔离，避免光环重绘触发吉祥物主体重绘

**动画时长标准**：

| 状态 | 持续时间 | 恢复目标 |
|------|----------|----------|
| happy | 1.5s | idle |
| celebrate | 3.0s | idle |
| thinking | 不定（跟随 AI 响应） | celebrate/sad |
| sad | 不自动恢复 | 等待下次交互 |
| curious | 2.0s | idle |

## 八、关键业务流程

### 8.1 应用启动流程

```
main() 
  → WidgetsFlutterBinding.ensureInitialized()
  → SharedPreferences.getInstance()  // 异步初始化
  → ProviderScope(
      overrides: [sharedPreferencesProvider.overrideWithValue(prefs)],
      child: LingxiApp(),
    )
  → LingxiApp.build()
      → ref.watch(themeModeProvider) / localeProvider / goRouterProvider
      → MaterialApp.router(theme, darkTheme, routerConfig)
      → goRouterProvider.redirect: 检查 onboarding_completed
          → 未完成: 重定向 /onboarding
          → 已完成: 重定向 /home
```

### 8.2 AI 对话主链路

```
用户输入
  → ChatController.sendMessage(text)
      1. 校验 AI Provider 可用性（currentAiProviderProvider）
      2. 读取 ProviderConfig 获取 temperature/maxTokens
      3. 若对话不存在则创建 Conversation
      4. 持久化用户消息到 MessageRepository
      5. 联动吉祥物 setAiThinking(true) → thinking
      6. PromptManager.getSystemPrompt(socraticMode) 注入系统提示词
      7. provider.chatStream(messages, options) 发起流式请求
      8. 监听事件：
         - TextDeltaEvent → 缓冲 + 50ms 节流刷新 UI
         - UsageEvent → 累加 token
         - DoneEvent → _commitAssistant() 落盘 + 吉祥物 celebrate
         - ErrorEvent → _finishStreamingWithError() + 吉祥物 sad
```

### 8.3 单节课学习闭环

```
LearningPathPage 点击课程
  → context.go('/learning/:courseId/:lessonId')
  → LessonPage 加载课程数据
  → PageView 轮播知识点
      每个知识点状态机：
        1. 学习卡片（_KnowledgePointPageWrapper）
        2. 测验（QuizWidget，正确率 ≥ 0.8 通过）
        3. 苏格拉底入口
        4. 苏格拉底对话（SocraticDialogPanel，至少 1 轮）
        5. 完成
  → _onKnowledgePointCompleted(kpId, score)
      → ProgressRepository.markCompleted(...)
      → MascotController.celebrate()
      → CelebrationService.sparkles/confetti
      → 判断全部完成 → 章节庆祝页
```

### 8.4 API Key 安全链路（**安全核心**）

```
用户输入 API Key
  → ProviderEditDialog._save()
  → ProviderConfigRepository.saveProvider(config)
      → 非密钥配置 JSON → SharedPreferences（键 provider_config_<type>）
      → apiKey → SecureStorageService.setApiKey() 加密存储
  → AiProvider 构造时从 SecureStorage 读取 apiKey 传入内存
  → dio 请求经 SecureLogInterceptor 自动脱敏（authorization/x-api-key 等）
  → 数据导出 ProviderConfig.toJson() 跳过 apiKey
  → assert(!json.containsKey('apiKey'))  // 不可移除的断言
```

### 8.5 Onboarding 引导流程

```
首次启动
  → goRouterProvider.redirect 检测 onboarding_completed=false
  → 重定向 /onboarding
  → 5 步 PageView（欢迎→自备 API→小犀彩蛋→学习路径→苏格拉底）
      每步切换触发 mascotController.setMood(curious/happy)
  → _complete() 持久化 onboarding_completed=true
  → 重定向 /onboarding/api-setup
  → 4 个 Provider 教程卡片
  → 用户选择 Provider → 分步教程
  → _finish() 进入主页 /home
```

### 8.6 吉祥物状态机

```
各页面通过 ref.read(mascotControllerProvider.notifier).setMood(...)
  → MascotController 状态更新
  → MascotWidget / RiveMascotWidget 订阅渲染
  → MascotPainter 作为 fallback（纯矢量绘制）

状态联动：
  - AI 思考中 → setAiThinking(true) → thinking
  - AI 完成 → celebrate() → celebrate (3s) → idle
  - AI 出错 → setMood(sad)
  - 用户点击 → triggerTap()：单次 happy 1.5s；2s 内连点 5 次 → celebrate 3s

全局浮层：
  - MascotOverlay 监听 mood != idle → 右下角悬浮显示
```

### 8.7 数据持久化链路

```
UI → Repository（业务语义化方法）
   → LingxiDatabase（Drift）
   → openConnection()
       → Android: sqflite
       → 桌面: sqlite3_flutter_libs FFI
   → NativeDatabase.createInBackground（后台 isolate）
   → storeDateTimeAsText: true（ISO8601 存储）
```

## 十一、安全红线与约束

> 🚨 **本章节为强制约束。任何修改若违反以下任一条，PR 将被直接拒绝。**

### 11.1 API Key 处理

- **API Key 只能通过 `SecureStorageService` 存储**（底层 `flutter_secure_storage`）
- **绝不**将 API Key 写入 Drift 数据库、`SharedPreferences`、文件、日志、导出 JSON
- `ApiKeys` 表只存元数据（providerType、baseUrl、model、temperature、maxTokens、enabled），**不含** apiKey 字段
- `ProviderConfig.apiKey` 字段仅在内存中持有，`toJson()` **必须**跳过 apiKey
- 构造 `AiProvider` 时从 `ProviderConfigRepository`（内部读 SecureStorage）获取 apiKey 传入

### 11.2 日志过滤

- **所有 dio 请求必须经过 `SecureLogInterceptor`**（`lib/features/ai/secure_log_interceptor.dart`）
- 自动脱敏字段：`authorization`、`x-api-key`、`x-goog-api-key`、`key`、`api_key`、`apiKey` 等 → 替换为 `[REDACTED]`
- **禁止**绕过 `SecureLogInterceptor` 直接 `print` 请求/响应内容
- 使用 `debugPrint` 而非 `print`（lint 规则 `avoid_print` 已启用）

### 11.3 数据导出

- `DataExportService.exportAll()` 中 `ProviderConfig.toJson()` **必须**不含 apiKey
- 代码中已有 `assert(!json.containsKey('apiKey'))` 断言，**不要移除**
- 导入时 `copyWith(apiKey: '')` 强制清空 apiKey
- 修改 `ProviderConfig.toJson` 时**必须**保持不含 apiKey 字段

### 11.4 .gitignore 必须包含敏感文件

```
*.keystore
*.jks
*.env
config.json
config.local.json
android/key.properties
macos/Runner/*.entitlements.priv
*.secure
```

### 11.5 截图/错误报告不能暴露 API Key

- 提交 Issue/PR 时，截图前确认无 API Key 残留
- 错误报告中若包含请求/响应，确保已通过 `SecureLogInterceptor` 脱敏
- **不要**在 Issue/PR 描述/commit message 中粘贴真实 API Key

### 11.6 不可修改的安全相关代码

除非任务明确要求，不要修改以下代码逻辑：
- `SecureStorageService`
- `SecureLogInterceptor` 的脱敏逻辑
- `ProviderConfig.toJson` 的字段过滤
- `DataExportService` 的 apiKey 断言
- `database.dart` 的 `storeDateTimeAsText`

### 12.2 测试约定

- 文件名：`xxx_test.dart`（与被测文件同名加 `_test` 后缀）
- 单元测试：`test('描述', () { ... })`
- Widget 测试：`testWidgets('描述', (tester) async { ... })`

### 12.3 内存数据库测试

```dart
final db = LingxiDatabase.forTesting(NativeDatabase.memory());
// 测试结束：await db.close();
```

- Repository 测试统一用 `NativeDatabase.memory()`，避免污染真实数据库
- SecureStorage 测试用 mock（见 `test/data/services/secure_storage_test.dart`）
- AI Provider 测试用 `http_mock_adapter` mock dio 请求

### 12.4 测试覆盖要求

- **Repository 必须有测试**（每个 Repository 至少覆盖 CRUD 主路径）
- **Service 必须有测试**（如 `StreakService`、`DataExportService`、`SecureLogInterceptor`）
- **AI Provider 必须有测试**（用 `http_mock_adapter` mock dio 请求）
- 新增功能时**必须**同步添加测试

### 12.5 性能预算

| 指标 | 目标 |
|------|------|
| 首屏渲染 | <1.5s |
| 流式首 token | <500ms（网络延迟除外） |
| 内存占用 | <200MB |
| 列表滚动 | 60fps |
| 动画帧率 | 60fps |
| 数据库查询 | <50ms |
| 课程加载 | <200ms |

**性能优化策略**：
- 流式响应使用 50ms 节流，避免频繁 setState
- 课程数据内存缓存，避免重复 IO
- 长列表使用 `ListView.builder` 懒加载
- 减少动效模式（`MediaQuery.disableAnimations`）时跳过所有动画
